Qu'est-ce qu'un maquettage d'interface ?
Un maquettage d'interface est une esquisse ou un modèle de l'interface utilisateur (UI) de l'application.

Il se concentre sur l'agencement des éléments (boutons, menus, champs de saisie, images, etc.) sans se soucier des détails visuels comme les couleurs ou les politiques.

Types de ma2. Types de maquettes :
Maquettes basse fidélité : Ce sont des croquis ou des schémas simples , généralement en: Ce sont des croquis ou des schémas simples, généralement en noir et blanc, pour représenter les structures de base et l'agencement des éléments.

Maquettes haute fidélité : Des maquettes plus détaillées, souvent interactives, qui montrent plus de détails visuels comme la couleur, les icônes et la disposition exacte.

## 2 différence prendre entre la différence entre : Zonage Maquette filaire Maquette Maquette

En résumé, le zonage est une première étape très abstraite qui définit les zones de l'interface, le wireframe (ou maquette filaire) est une maquette plus détaillée montrant la structure sans design, tandis que la maquette (ou maquette haute fidélité) est la version finale visuellement détaillée de l'interface.

## 3 Choix d'un outil de maquettage

Figma : Outil de design et prototypage collaboratif en ligne, idéal pour les équipes qui travaillent à distance. Très populaire pour le design UI/UX.

Sketch : Outil de design vectoriel pour Mac, spécialisé dans la création d'interfaces utilisateur (UI).

Adobe XD : Outil de design et prototypage avec intégration Adobe, adapté pour des prototypes interactifs simples.

Axure RP : Outil puissant pour créer des prototypes interactifs et fonctionnels complexes.

Marvel : Outil simple pour des prototypes rapides et interactifs.

InVision : Outil de prototypage interactif collaboratif, idéal pour partager des prototypes avec des équipes.

Balsamiq : Outil de wireframing pour créer des maquettes basse fidélité rapidement.

Proto.io : Outil de prototypage interactif en ligne, facile à utiliser pour des projets web et mobiles.

Framer : Outil avancé pour des prototypes interactifs avec animations et intégration de code.

## 4 Encadrer la demande client

Objectifs et Contexte : Comprendre le but du projet, les problèmes à résoudre et le contexte (secteur, tendances, etc.).

Besoins et Attentes : Identifier les besoins fonctionnels et non fonctionnels (fonctionnalités, performance, design, etc.), ainsi que les attentes qualitatives.

Contraintes : Définir les contraintes techniques, légales et de design (compatibilité, réglementation, etc.).

Livrables et Méthodologie : Préciser les livrables attendus, les étapes du projet (conception, développement, tests) et la méthodologie utilisée (agile, waterfall).

Communication : Clarifier les points de contact, la fréquence des réunions et les modalités de validation.

Risques et Problèmes : Anticiper les risques (techniques, fonctionnels) et les imprévus possibles.

Calendrier et Budget : Définir un planning avec des dates clés et estimer le budget disponible.

Validation : Vérifier que la demande est bien comprise et formaliser cela dans un document de cadrage.

## 5 Découvrir les règles ergonomiques et d'expérience utilisateur

Les règles ergonomiques visent à garantir que l'interface est facile à utiliser, claire et agréable. Les principes UX se concentrent sur la satisfaction de l'utilisateur en tenant compte de ses 
attentes et besoins. L’objectif est de rendre l'interaction fluide, intuitive et sans frustration, tout en respectant les normes et les bonnes pratiques du design. Pour cela, il est important 
d’avoir une interface claire, réactive, accessible et flexible, tout en mettant en place un processus de tests réguliers pour garantir une amélioration continue.

# HTML

## 1 Découverte de la sainte trinité du web (HTML, CSS et JS)

    HTML : Structure la page avec des balises de contenu.

    CSS : Style et met en forme le contenu (polices, couleurs, disposition).

    JavaScript : Ajoute de l’interactivité, des animations et de la dynamique à la page.

Ces trois technologies sont les bases fondamentales du développement web moderne. En combinant HTML pour la structure, CSS pour le style et JavaScript pour l'interactivité, vous pouvez créer des sites web riches et réactifs.

## 2 Découvrez les bases de HTML :

Le HTML est essentiel pour construire des pages web, et apprendre ses bases permet de structurer correctement le contenu de manière logique. Une fois que vous maîtrisez le HTML de base, vous pouvez intégrer des éléments interactifs avec du JavaScript et styliser vos pages avec du CSS.

Si vous êtes débutant, il est important de pratiquer et de tester différentes balises pour voir leur effet sur la page. Vous pouvez utiliser un éditeur de texte comme Visual Studio Code, Sublime Text ou même des outils en ligne comme CodePen pour tester votre code HTML en temps réel.

N’hésitez pas à explorer des concepts plus avancés une fois que vous êtes à l’aise avec ces bases !

## 3 Comprendre la différence entre les liens relatifs et les liens absolus

    Lien absolu : Spécifie toute l'URL complète avec le domaine, le protocole et le chemin.

        Exemple : https://www.exemple.com/chemin/vers/la/ressource

        Utilisé pour les liens externes.

    Lien relatif : Spécifie le chemin par rapport à la page actuelle sans inclure le domaine.

        Exemple : page2.html ou ../page2.html

        Utilisé pour la navigation interne au sein d’un même site web.

Les liens relatifs sont préférés pour les sites internes, tandis que les liens absolus sont utilisés pour les ressources externes ou lorsque vous avez besoin de spécifier une URL complète.
